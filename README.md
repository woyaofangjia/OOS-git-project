# 校园二手交易平台 README

## 1. 项目概述

本项目是一个C2C校园二手交易平台，旨在为在校学生提供一个便捷、安全的二手商品交易环境。平台支持商品发布、求购信息发布、购物车管理、个人信息管理等功能，采用现代化的前后端分离架构，提供优秀的用户体验。

## 2. 技术栈

- **后端技术**：Java 21, Spring + SpringMVC + MyBatis (SSM框架), Thymeleaf模板引擎
- **前端技术**：HTML5, CSS3, JavaScript, jQuery, EasyUI
- **数据库**：MySQL
- **其他技术**：WebSocket, MD5加密, 敏感词过滤, 图片处理

## 3. 系统设计

### 3.1 需求分析

校园二手交易平台需要满足以下核心需求：
- 提供商品发布和浏览功能
- 支持求购信息发布和管理
- 实现购物车和订单管理
- 提供用户注册、登录和个人信息管理
- 实现站内搜索引擎
- 确保系统安全性和稳定性

### 3.2 概要设计

#### 3.2.1 系统模块

- **用户模块**：注册、登录、个人信息管理
- **商品模块**：商品发布、商品浏览、商品详情、商品分类
- **求购模块**：求购信息发布、求购信息浏览
- **交易模块**：购物车、订单管理
- **搜索模块**：站内搜索引擎
- **消息模块**：商品留言、求购留言
- **管理模块**：我的发布、我的求购、已购买商品

## 4. 实现过程

### 4.1 商品首页实现

负责显示热门商品信息和网站导航。导航栏支持用户登录/注册状态切换，已登录用户可查看个人信息、发布商品等。通过Ajax获取热门商品数据，实现轮播图展示和商品卡片显示。

首页设计要点：
- 顶部导航栏：Logo、搜索框、用户状态区
- 轮播图：展示热门商品，支持自动切换和手动控制
- 商品展示区：以卡片形式展示其他热门商品

### 4.2 二手商城实现

展示所有二手商品，提供类别筛选和分页功能。侧边栏实现三级分类展示，用户可通过点击类别查看对应商品。

实现要点：
- 三级分类侧边栏：支持鼠标悬停展示子分类
- 商品列表：分页展示商品卡片
- 分页控件：支持上下页切换和跳转到指定页

### 4.3 商品详情实现

显示商品的详细信息，包括图片、名称、价格、数量等，并提供留言和加入购物车功能。

实现要点：
- 商品信息展示：图片轮播、价格、数量等
- 留言板：用户可查看和发表留言
- 加入购物车：一键添加到购物车

### 4.4 站内搜索引擎实现

每个页面顶部都有搜索框，支持模糊搜索商品信息。

实现要点：
- 搜索输入框：支持关键词输入
- 搜索结果页：分页展示符合条件的商品

### 4.5 发布商品实现

用户可输入商品信息并上传图片，通过Ajax提交到后台进行处理。

实现要点：
- 三级联动分类选择
- 图片上传和预览
- 表单验证和提交

### 4.6 发布求购信息实现

类似于发布商品功能，但针对求购需求。

实现要点：
- 三级联动分类选择
- 图片上传和预览
- 求购价格和数量设置

### 4.7 个人信息页面实现

展示和修改用户的个人信息，如用户名、真实姓名、宿舍号等。

实现要点：
- 信息展示：使用Accordion组件
- 信息修改：表单提交和Ajax局部刷新

### 4.8 我发布的商品实现

显示用户已发布的商品，支持修改和下架操作。

实现要点：
- 商品列表展示
- 修改功能：可编辑商品信息
- 删除功能：可下架商品

### 4.9 我发布的求购信息实现

显示用户已发布的求购信息，支持修改和删除操作。

实现要点：
- 求购信息列表展示
- 修改功能：可编辑求购信息
- 删除功能：可下架求购信息

### 4.10 购物车实现

展示用户添加到购物车的商品，支持数量修改、全选/取消全选和结算功能。

实现要点：
- 商品列表：显示商品信息和数量
- 总价计算：实时更新选中商品总价
- 结算功能：选择收货地址和支付方式

### 4.11 登录注册实现

支持用户登录和注册，注册时需要通过手机验证码验证。

实现要点：
- 登录界面：手机号和密码登录
- 注册界面：手机号验证、密码设置
- 密码找回：通过手机号验证码重置密码
- 3D立方体切换效果：提升用户体验

## 5. 后台实现细节

### 5.1 后台结构

- **java目录**：存放Java代码
  - **bean**：用于序列化到前端的实体类
  - **controller**：Spring MVC控制器，处理请求
  - **dao**：与数据库交互的接口
  - **pojo**：普通Java实体类
  - **service**：业务逻辑层，连接controller和dao
  - **token**：生成唯一标识，防止重复提交
  - **tool**：工具类，如文件处理、敏感词过滤等
- **resources目录**：存放资源文件
  - MyBatis映射文件
  - Spring配置文件
  - Spring MVC配置文件
- **static目录**：存放静态资源
  - CSS样式文件
  - JavaScript文件
  - 图片资源
- **templates目录**：存放Thymeleaf模板文件

### 5.2 关键技术实现

1. **单用户登录**：使用WebSocket和Spring WebSocket实现，确保同一账号只能在一处登录

2. **数据安全**：
   - 密码加密：使用MD5加密
   - 防止重复提交：使用Token机制
   - 敏感词过滤：通过文本匹配过滤敏感内容
   - 图片审核：使用百度开发者平台进行色情图片识别

3. **性能优化**：
   - 图片压缩：先加载压缩图片，鼠标悬停时显示高清图片
   - 工厂模式：优化工具类的创建和使用
   - 数据库索引：提高查询效率

4. **安全防护**：
   - XSS防护：使用消毒和HttpOnly
   - SQL注入防护：参数绑定和错误处理
   - CSRF防护：表单token、验证码和Referer check
   - 敏感信息加密：单向散列表加密和对称加密

## 6. 问题分析与解决方案

### 6.1 单用户登录问题

**问题**：初始设计中使用两个Map存储登录信息，导致线程安全问题。
**解决方案**：使用一个Map存储手机号码和对应时间戳，通过比较时间戳判断是否有其他用户登录。

### 6.2 工具类设计问题

**问题**：多用户登录时可能导致线程不安全或内存占用过高。
**解决方案**：采用静态工厂模式-静态内部类，将所有工具类集中到一个总类中。

### 6.3 MyBatis多参数问题

**问题**：多参数传递时出现参数未定义的错误。
**解决方案**：将多个参数封装为Map再传递给MyBatis处理方法。

### 6.4 手机验证码发送问题

**问题**：邮箱验证被识别为垃圾邮件，无法正常发送。
**解决方案**：利用139邮箱发送验证码，手机用户绑定后可收到短信通知。

### 6.5 Ajax局部刷新事件绑定问题

**问题**：新显示的内容无法绑定点击事件。
**解决方案**：在Ajax的for循环显示结果内容后，为按钮等绑定相应事件。

## 7. 用户使用说明

### 7.1 首页导航

访问网站首页，可浏览热门商品，通过轮播图了解推荐商品，点击商品卡片查看详情。

### 7.2 游客权限

普通游客可浏览首页、商城和求购商城，但发布商品、求购信息等操作需要先登录。

### 7.3 注册与登录

点击右上角注册按钮，输入手机号获取验证码，设置密码后即可完成注册。忘记密码时可通过手机号验证码重置。

### 7.4 商城使用

点击导航栏"商城"进入，可通过左侧分类筛选商品，或使用搜索框进行模糊查询。每页显示12个商品，可通过分页控件浏览更多。

### 7.5 求购商城

点击导航栏"求购商城"进入，可浏览其他用户的求购信息。

### 7.6 发布信息

登录后，将鼠标悬停在用户名上，从下拉菜单中选择"发布商品信息"或"发布求购信息"，按提示填写并提交。

### 7.7 管理我的发布

从用户下拉菜单中选择"我发布的商品"或"我的求购商品"，可查看、修改或删除已发布的信息。

### 7.8 购物车管理

浏览商品时点击"添加至购物车"，从用户下拉菜单中进入"我的购物车"，可修改数量、删除商品或进行结算。

## 9. 开发环境配置指南

### 9.1 环境要求

- **JDK**: 21 版本
- **Maven**: 3.9.10 版本
- **MySQL**: 8.0.40 版本
- **Jetty**: 9.4.53 版本（通过Maven插件提供，无需额外安装）
- **IDE**: 推荐使用VSCode、IntelliJ IDEA或Eclipse

### 9.2 开发环境配置

#### 9.2.1 安装必要软件

1. 安装JDK 21并配置环境变量
2. 安装Maven 3.9.10并配置环境变量
3. 安装MySQL 8.0.40并启动服务

#### 9.2.2 导入数据库

1. 创建名为`Campus-SecondHand`的数据库
2. 执行`SQL/校园二手交易平台数据库1.0.sql`文件导入数据结构和初始数据
3. 确保数据库用户名和密码与`jdbc.properties`中的配置一致（默认：用户名`root`，密码`123456`）

#### 9.2.3 配置数据库连接

如果需要修改数据库连接信息，请编辑`src/main/resources/jdbc.properties`文件：

```properties
driver=com.mysql.jdbc.Driver
url=jdbc:mysql://localhost:3306/Campus-SecondHand
username=root
password=123456
```

### 9.3 运行项目的方法

#### 9.3.1 方法一：使用Maven命令直接运行（推荐开发时使用）

1. 打开命令行工具，进入项目根目录
2. 执行以下命令编译并运行项目：

```bash
mvn jetty:run
```

3. 等待服务启动完成后，访问 http://localhost:8080 即可

1. 直接双击运行`start-dev.bat`文件
2. 脚本会自动检查环境、编译项目并启动Jetty服务
3. 等待服务启动完成后，访问 http://localhost:8080 即可

#### 9.3.3 方法三：使用WAR包运行（生产环境推荐）

1. 编译项目生成WAR包：

```bash
mvn clean package
```

2. 使用Jetty运行WAR包：

```bash
mvn jetty:run-war -Dwar.path=target/ROOT.war
```

3. 或者直接运行`deploy.bat`脚本，它会自动打包并启动服务
4. 访问 http://localhost:8080 即可

### 9.5 常见问题及解决方案

#### 9.5.1 端口被占用

**问题**：启动时提示8080端口已被占用

**解决方案**：
- 使用`netstat -ano | findstr :8080`查看占用端口的进程
- 使用`taskkill /F /PID 进程ID`终止占用端口的进程
- 或修改`pom.xml`中Tomcat插件的端口配置

#### 9.5.2 数据库连接失败

**问题**：启动时提示无法连接数据库

**解决方案**：
- 检查MySQL服务是否启动
- 确保数据库名、用户名和密码正确
- 检查数据库配置文件`jdbc.properties`中的连接信息

#### 9.5.3 编译失败

**问题**：`mvn compile`或`mvn jetty:run`命令执行失败

**解决方案**：
- 确保使用JDK 21版本
- 检查JDK和Maven是否正确安装

### 9.6 开发建议

1. **代码风格**：遵循项目现有的代码风格和命名规范

3. **日志记录**：使用log4j记录重要操作和错误信息
4. **异常处理**：统一的异常处理机制已在web.xml中配置，自定义异常可映射到/error路径
5. **静态资源**：CSS、JavaScript和图片等静态资源放在webapp目录下
